继教交叉考勤缓存清理（操作说明）

适用场景：数据库已删除学员考勤记录，但仍提示「请勿在同一天内考勤多个项目」或「同时间内无法参加多个项目考勤」，多为 Redis 残留导致。
步骤1：确认报错类型
· 「请勿在同一天内考勤多个项目」→ 清理当天同套交叉（下方步骤2）
· 「同时间内无法参加多个项目考勤」→ 清理周期交叉（下方步骤3）
· 「请勿重复考勤」→ Key：attSpaceTime-{downId}，Field：人员ID（整Key删除会影响该周期所有人间隔记录，建议用 HDEL）
· 「本批次已考勤」→ Key：attendanceSet-{batchId}（Set 类型，需 redis-cli 执行 SREM，本页暂不支持）
步骤2：清理当天同套交叉（使用下方「Hash 字段删除」一行）
Hash 的 Key：projStandPerson-{cmeStandardKindId}（套ID）
Field：{comPersonId}（学员人员ID）
勿对整Key删除，否则会清空该套下所有学员当天交叉记录。
步骤3：清理周期交叉（使用下方「Hash 字段删除」一行）
Hash 的 Key：{listOrder}-{人员ID首字符}（如 listOrder=1、人员ID 以 8 开头则为 1-8）
Field：{comPersonId}（完整人员ID）
listOrder 为省份序号，可在项目周期信息或 Redis attendanceInfo-{downId} 的 province 字段查看；不确定时可尝试 1、2 等常见值。
