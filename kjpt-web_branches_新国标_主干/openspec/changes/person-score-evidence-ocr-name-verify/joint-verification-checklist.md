# OCR 联调检查清单（6.1 ~ 6.5）

> 适用范围：单位/科室/个人三条录入链路（录入页 + 管理页编辑提交）。

## 0. 预置准备

- 准备 3 组附件：
  - `MATCH`：附件中包含与人员姓名一致文本（可含空格/全角）
  - `NO_TEXT`：纯图形或模糊附件（OCR 无可识别文字）
  - `MISMATCH`：附件中仅包含其他姓名
- 准备 1 组超限或非法格式附件（用于验证上传限制）。
- 准备 1 条可用于审核通过的数据（用于验证红标解除）。

## 1. 开关=0（对应 6.1）

- 在配置中将 `fun_singleproj_evidence_ocr_verify=0`。
- 分别走通：
  - 单位：`grant_score_singleproj.html` 或 `grant_score_manage.html`
  - 科室：`dept_score_singleproj.html` 或 `dept_score_manage.html`
  - 个人：`individual_score_singleproj.html` 或 `individual_score_manage.html`
- 期望：
  - 提交成功
  - Network 中无 `.../cmeSingleprojScore/evidenceOcrVerify` 请求
  - 业务保存接口正常返回

## 2. 开关=1 三态与弹框（对应 6.2）

- 将 `fun_singleproj_evidence_ocr_verify=1`，逐角色验证。
- `MATCH` 附件：
  - 期望可直接提交，保存成功。
- `NO_TEXT` 附件：
  - 期望提示“无法识别，请重新上传清晰图片”
  - 本次提交被阻断，不触发保存接口。
- `MISMATCH` 附件：
  - 第 1、2 次提交：弹窗出现（“重新上传 / 忽略并继续录入”）
  - 点“重新上传”：提交终止，附件列表清空
  - 点“忽略并继续录入”：继续保存，记录异常标记
  - 第 3 次提交：不弹窗，自动按异常继续保存

## 3. 空格/全半角匹配（对应 6.3）

- 人员姓名使用 `张三`，附件中仅出现 `张 三` 或全角变体。
- 期望：
  - OCR 结果判定为 `MATCH`
  - 可正常保存，无误拦截

## 4. 审核通过解除标红（对应 6.4）

- 选择一条被标记异常的数据，进入审核列表。
- 期望：
  - 审核前在列表中存在红底/叹号异常标识
  - 审核通过后刷新列表，异常标识消失
  - 驳回流程无新增通知副作用

## 5. 三角色完整链路（对应 6.5）

- 单位、科室、个人各完成 1 条“上传附件 -> OCR校验 -> 保存 -> 上报/审核”全链路。
- 期望：
  - 三角色行为一致
  - 页面提交与管理页编辑均遵循同一 OCR 规则
  - 异常数据可在审核链路中持续追踪，审核通过后恢复正常
