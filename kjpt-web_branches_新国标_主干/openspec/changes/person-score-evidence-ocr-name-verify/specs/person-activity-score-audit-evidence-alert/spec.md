## ADDED Requirements

### Requirement: 异常学分在审核列表整行标红

当学分因「忽略并继续录入」或第 3 次自动提交而被标记为姓名核验异常，且仍处于待审核状态时，个人活动学分审核模块中该条记录 MUST 以红色样式整行高亮展示，与正常学分有明显视觉区分。

#### Scenario: 异常学分待审列表展示

- **WHEN** 审核人员打开个人活动学分审核列表且某条记录 `evidence_name_verify_status` 为异常
- **THEN** 该行以红色高亮样式展示

### Requirement: 佐证列红色叹号角标与 Tooltip

对上述异常学分，佐证材料查看列 MUST 展示红色叹号小角标（!）。鼠标悬停于角标时，系统 MUST 展示 Tooltip，文案为：「该佐证材料照片中疑似未包含学分归属人姓名，请您关注并查验」。

#### Scenario: 悬停叹号角标

- **WHEN** 审核人员将鼠标悬停于异常学分的佐证列叹号角标
- **THEN** 显示上述 Tooltip 文案

### Requirement: 审核通过后解除标红与角标

当审核人员通过现有审核流程对该条学分审核通过时，系统 MUST 自动清除姓名核验异常标记，整行红色高亮与佐证列叹号角标 MUST 同步解除，恢复为正常展示样式。

#### Scenario: 审核通过

- **WHEN** 审核人员对异常学分执行审核通过操作成功
- **THEN** 列表再次加载时该行无红色高亮且无叹号角标

### Requirement: 审核驳回不通知录入人

审核人员对上述学分执行审核驳回时，系统 MUST NOT 向学分录入人员发送任何新增通知（相对变更前，不增加 OCR 相关通知渠道）。

#### Scenario: 审核驳回

- **WHEN** 审核人员驳回一条姓名核验异常的学分
- **THEN** 不向录入人发送与本需求相关的通知
